## 产品概述

将年度检查报告自动填充系统的数据流重构为统一中间数据结构A（UnifiedReportData），实现模板分析、取数、填数、校验四个环节共享同一份纯数据层结构，并在前端完整展示填充结果。

## 核心功能

- **模板分析**产出 `TemplateStructure`，描述模板中每个 sectionId 对应的占位符字段映射、表格列定义、键值对标签清单、签名行位置
- **取数阶段**：`.rj` 知识图谱解析器（statements-parser）和 `原始MD.docx` 章节切分器（chapter-extractor）均产出 `UnifiedReportData`（纯字符串/对象，不含 Word XML）
- **双源合并**：将 .rj 和 MD 的 `UnifiedReportData` 按 sectionId 合并，MD 优先提供键值对和签名数据，.rj 优先提供列表型表格数据
- **填数阶段**：从 `UnifiedReportData` 读取数据，结合 `TemplateStructure` 定位模板位置，通过 xml-subtree-inserter 写入 Word
- **数据校验**：基于 `UnifiedReportData` + `ValidationRule[]` 进行必填字段完整性、日期格式、数值范围等规则校验
- **前端预览**：渲染 `UnifiedReportData` 的完整内容——基本信息卡片 + 每章节的键值对表、列表型表格（全行数据含折叠/展开）、签名数据面板
- 旧类型 `Chapter`、`SourceAnalysis`、`TablePreview` 标记 `@deprecated` 保留兼容，新路由逐步启用

## 技术栈

- 后端：Node.js + TypeScript + Express
- Word 操作：XML 直接操作（不引入 docx-js）
- 解析：JSON-LD 解析器（statements-parser）、Word XML 章节切分（chapter-extractor）
- 前端：原生 HTML/CSS/JS

## 实现方案

### 总体策略

三阶段渐进重构，每步可单独编译验证，不破坏现有功能：

1. **第一阶段**：新增类型定义 + TemplateStructure 分析服务
2. **第二阶段**：改造取数层（parsers → UnifiedReportData）+ 新建 data-merger
3. **第三阶段**：改造填充层（filler + xml-subtree-inserter）+ 校验引擎 + 前端预览 + API 适配

### 关键决策

1. **数据层与模板层彻底分离**：`UnifiedReportData` 只存数据值（字符串key-value），`TemplateStructure` 存模板结构信息。填充时两者结合：`TemplateStructure` 定位模板位置，`UnifiedReportData` 提供填充值。

2. **xml-subtree-inserter 新增重载**：保留原有接受 XML 字符串的方法，新增接受 `DataTable`/`KeyValuePair` 纯数据参数的重载，内部将纯数据转为单元格文本再写入，无需解析 XML。

3. **渐进式迁移**：旧 `SourceAnalysis` / `Chapter` 标记 `@deprecated`，`/fill-by-copy` 同时支持新旧两种数据流。新服务独立文件，避免大规模重构风险。

4. **前端预览直接消费 UnifiedReportData**：替换原有 `renderPreview(analysis)` 中的 `analysis.tablePreviews`（仅列表型表格），新增渲染 `sections[].kvPairs`（键值对表）和 `sections[].signature`（签名数据），提供更完整的数据预览。

### 实现要点

- 性能：合并时用 Map 按 sectionId 索引 O(1)；签名行填充只扫描每章节最后一行
- 兼容性：保留旧 `mergeSourceAnalysis` 方法标记 `@deprecated`，`/fill-by-copy` 路由通过 `useNewFlow` 参数切换新数据流
- 回归控制：验证脚本 verify-dual-source.ts 同时测试新旧路径

## 关键代码结构

### 核心新类型（全部新增于 server/src/types/index.ts）

```typescript
// ---- 统一报告数据（纯数据层，不含任何 Word XML）----
interface UnifiedReportData {
  basicInfo: BasicInfo;
  sections: SectionData[];
}
interface SectionData {
  id: string;                     // 章节编号 "1-1""2-3"
  title: string;
  kvPairs: KeyValuePair[];        // 键值对列表
  tables: DataTable[];            // 列表型表格
  signature: SignatureBlock;      // 签名数据
}
interface KeyValuePair { key: string; value: string; }
interface DataTable {
  tableType: string;              // 实体类型标识
  headers: string[];              // 列名（中文表头）
  rows: Record<string, string>[]; // 每行 header→value 映射
}
interface SignatureBlock {
  inspectorName: string; inspectorDate: string;
  checkerName: string; checkerDate: string;
  reviewerName: string; reviewerDate: string;
}

// ---- 模板结构定义 ----
interface TemplateStructure {
  sections: TemplateSection[];
}
interface TemplateSection {
  sectionId: string;
  placeholderFields: { mapTo: string; pattern: string }[];
  tables: { tableIndex: number; isKeyValue: boolean; kvKeys?: string[]; columns?: { header: string }[] }[];
  signaturePosition: { tableIndex: number } | null;
}

// ---- 校验 ----
interface ValidationRule {
  scope: { sectionId?: string; tableType?: string };
  type: "required" | "format" | "range" | "consistency";
  field: string;
  config?: { pattern?: string; min?: number; max?: number };
}
interface ValidationReport {
  passed: boolean;
  errors: { sectionId: string; field: string; message: string }[];
}
```

## Agent Extensions

### Skill

- **verification-before-completion**
- Purpose: 在每阶段完成后运行 verify-dual-source.ts 验证端到端数据流，确认重构未破坏现有功能，并验证前端预览渲染正确
- Expected outcome: 编译通过 + 新旧路径均验证脚本成功 + 前端预览面板正确展示 UnifiedReportData

- **systematic-debugging**
- Purpose: 改造期间如出现类型错误、数据流断裂或前端渲染异常，快速定位根因
- Expected outcome: 明确问题链路，提供修复方案
