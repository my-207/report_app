## 问题描述

列表型表格填充逻辑不符合用户需求，需要修复：

1. **取数逻辑**：当前 `extractTableDataRows` 跳过表头后取所有剩余行，没有跳过签名行。应改为取第2行~倒数第2行，跳过最后一行签名行。

2. **写入逻辑**：当前 `fillTableFromSource` 通过 `isRowEmpty` 找空行填入，签名行被当作普通行处理。应改为：在第2行位置插入空白行填入数据，签名行单独处理。

3. **签名行处理**：当前无特殊处理，检测/校对信息被忽略。应从源文档最后一行提取签名数据，填入模板签名行对应单元格。

## 修复目标

修改 `xml-subtree-inserter.service.ts` 两个方法：

- `extractTableDataRows`：返回 `{dataRows, signatureRow}`，跳过表头和签名行
- `fillTableFromSource`：用模板第2行作为数据行模板逐行生成数据行，用模板最后一行作为签名模板填入签名数据，组装为 `表头 + 数据行 + 签名行`

## 技术方案

### 修改文件

`server/src/services/xml-subtree-inserter.service.ts` — 仅修改 `fillTableFromSource` (23-89行) 和 `extractTableDataRows` (92-112行)。

### 实现细节

**extractTableDataRows** 新签名：

```typescript
private extractTableDataRows(tableXml: string): {
  dataRows: string[][];
  signatureRow: string[] | null;
}
```

- 收集所有行（跳过表头第一行）
- 判断最后一行是否为签名行（包含"检测/校对/审核/检查/审批"关键词）
- 是签名行 → dataRows = 去掉最后一行，signatureRow = 最后一行
- 否 → dataRows = 全部，signatureRow = null

**fillTableFromSource** 新逻辑：

```
1. 定位模板表格（不变）
2. extractTableDataRows(sourceTableXml) → {dataRows, signatureRow}
3. extractAllRows(targetTbl) → targetRows[]
4. headerRow = targetRows[0]（表头，保留不变）
5. dataRowTemplate = targetRows[1].xml（第2行作为数据行格式模板）
6. lastRow = targetRows[-1]（签名行模板）
7. for each srcRow in dataRows:
     filledRow = fillRowWithData(dataRowTemplate, srcRow)
8. filledSignatureRow = signatureRow ? fillRowWithData(lastRow.xml, signatureRow) : lastRow.xml
9. rebuiltTbl = headerRow.xml + filledDataRows.join("") + filledSignatureRow
10. 替换回 targetXml
```

- `fillRowWithData`、`extractCellTexts`、`extractAllRows` 完全复用，无需修改
- 签名行通过 `fillRowWithData` 按列顺序填入检测/校对人员信息
