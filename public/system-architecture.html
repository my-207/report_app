🔄 核心数据流

1 用户上传 Word 报告模板 (.docx) + 数据源 (.rj / 原始MD.docx)
2 模板解包 → XML 结构提取 → 占位符 & 表格分析
3 .rj 知识图谱解析 → 实体提取 → 属性映射 → 章节表格构建
4 原始MD.docx 解析 → 键值对提取 → 签名行提取 → 章节分组
5 双源数据合并 → 表头模糊匹配 → 模板-数据表格对应
6 XML 直接填充：占位符替换 → 键值对填充 → 列表行填充 → 签名行填充
7 数据校验 → 异常字段标注 → 生成校验报告
8 XML 打包 → 生成 Word 报告 → 下载 / 推送 + 异常标注清单

📡 API 接口矩阵

POST /api/upload-template — 上传 .docx 模板，返回 sessionId
POST /api/upload-rj — 上传 .rj 知识图谱，返回解析预览
POST /api/upload-source-docx — 上传原始MD.docx 作为补充源
POST /fill-by-copy — 核心：双源子树复制填充，生成报告
GET /api/download/:file — 下载生成的年度检查报告
GET /api/employee/info — 数字员工元数据查询
GET /api/health — 健康检查：状态、内存、任务数
POST /api/task/execute — 异步任务执行 + Webhook 回调
